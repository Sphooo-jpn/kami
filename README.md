# 紙 — kami

**依存ゼロ・ビルド不要のブラウザ内 WYSIWYG PDF エディタ。**
ページ上に文字・図形・画像を自由に配置し、**自前実装の PDF ジェネレータ**で本物の PDF に書き出します。npm パッケージも CDN も一切使いません（`pdf.js` も `pdf-lib` も不使用）。

## 特徴

- 📄 **本物の PDF を出力** — PDF のバイナリ構造（オブジェクト・xref・コンテンツストリーム）を素の JavaScript で組み立て。標準14フォントによる**選択・コピー可能なテキスト**、ベクター図形、JPEG 画像埋め込みに対応。
- 🈶 **日本語もそのまま** — WinAnsi で表せない文字（日本語・絵文字など）を含むページは、Canvas で高解像度（約180dpi）にラスタライズして画像として埋め込むため、**見た目どおりに出力**されます。
- ✍️ **WYSIWYG 編集** — テキスト（フォント/サイズ/太字/斜体/下線/色/揃え/行間）、長方形（角丸）、楕円、直線、画像。ドラッグで移動、ハンドルでリサイズ、スナップガイド、重ね順、ページ内整列。
- 🗂 **複数ページ / 用紙サイズ** — A4・Letter・A5・A3・B5・Legal・正方形・名刺、縦横切替、背景色、任意サイズ（pt）。
- ↩️ **Undo/Redo・自動保存** — 変更は `localStorage` に自動保存。プロジェクトは `.kami.json` で書き出し／読み込み。
- 🌗 **ダーク/ライトテーマ**、キーボードショートカット、拡大縮小。
- 🔒 **完全ローカル** — データは端末内にのみ保存され、**外部送信は一切ありません**。

## 書き出しモード

| モード | 中身 | 用途 |
|---|---|---|
| **スマート** | 欧文だけのページは選択可能なベクター、日本語等を含むページは高品質画像 | 迷ったらこれ |
| **ベクター（軽量）** | 全ページをベクターで（文字は選択可、ファイル小） | 欧文中心の資料 |
| **高品質画像** | 全ページを画像化（見た目を完全再現） | フォント・効果を厳密に保ちたい時 |

## 使い方（WSL / Windows）

```bash
cd ~/personal_dev/kami
./kami            # WSL から Windows の既定ブラウザで開く
```

`./kami` は `index.html` を既定ブラウザで開くだけです。ビルドもサーバも不要。
どの OS でも、単に `index.html` をブラウザで開けば動きます。

### 基本操作

1. 左のツール（選択 / 文字 / 四角 / 丸 / 線 / 画像）を選ぶ
2. ページ上を**ドラッグ**して作成（クリックでも既定サイズで配置）
3. テキストは**ダブルクリック**で編集、右パネルで見た目を調整
4. 右上の **PDF 書き出し** から出力

### ショートカット

| キー | 動作 |
|---|---|
| `V` `T` `R` `O` `L` `I` | ツール切替（選択/文字/四角/丸/線/画像） |
| `Ctrl+Z` / `Ctrl+Shift+Z` | 元に戻す / やり直し |
| `Ctrl+D` | 複製 |
| `Del` / `Backspace` | 削除 |
| `↑ ↓ ← →` | 1pt 移動（`Shift` で 10pt） |
| `Ctrl+S` | プロジェクト保存（`.kami.json`） |
| `Ctrl` + `+ / − / 0` | 拡大 / 縮小 / フィット |
| 角のハンドル + `Shift` | 縦横比を保ってリサイズ |

## 仕組み

モデル（ページと要素、単位は pt・左上原点）を 3 つのレンダラで共有します。

- **DOM** … 編集ビュー（`.sheet` 上に絶対配置、図形はインライン SVG）
- **Canvas** … ラスタ書き出し／将来のサムネイル用（`renderPageToCanvas`）
- **PDFWriter** … ベクター書き出し（`pdfwriter.js`。欧文は選択可能テキスト）

`pdfwriter.js` は DOM 非依存の純粋モジュールなので、Node 単体でテストできます。

## テスト

```bash
bash test/run.sh
```

- **(1) Node**: `pdfwriter.js` が生成する PDF の構造（ヘッダ・xref オフセットの正確さ・ストリーム長・WinAnsi 判定など）を検証。
- **(2) ブラウザ**: ヘッドレス Chrome で実際の `measureText`／Canvas／書き出しを走らせ、折返し・ベクター/ラスタ振り分け・多ページ PDF 生成までを検証。

（このリポジトリの PDF 出力は `pypdf` でも問題なくパースでき、欧文ページはテキスト抽出可能、日本語ページは埋め込み画像として正しく再現されることを確認済みです。）

## スコープと限界（正直なところ）

- kami は PDF を**作る／組む**ための WYSIWYG エディタです。既存の任意 PDF を読み込んでページを再編集する機能は、依存ゼロ方針（PDF パーサ/レンダラを積まない）のため**対象外**です。スキャン画像等を取り込んで注釈を載せる用途は画像配置で対応できます。
- ベクター出力のフォントは標準14フォント（Helvetica / Times / Courier 系）。画面上の日本語はシステムフォントで表示され、出力時は自動で画像化されます。
- 文字幅の計測はブラウザの `measureText`（Arial 等）に基づくため、ベクター出力（Helvetica 実体）と 1 行内でごく僅かな字送り差が出ることがあります。厳密な見た目再現が必要なら「高品質画像」をどうぞ。
